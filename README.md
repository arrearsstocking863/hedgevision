## 🔧 Troubleshooting

### The app does not open

Try these steps:

- right-click the file and choose Run as administrator
- check that Windows did not block the file
- make sure you extracted the ZIP file first
- download the file again from the release page

### Windows says the file is unsafe

This can happen with new apps from GitHub.  
If the file came from the release page linked above, try the following:

- open the file properties
- check whether Windows blocked it
- allow the app to run
- start it again

### The screen is blank

If the app opens but shows little or no data:

- check that you loaded a file
- confirm the file has the right columns
- make sure the date values are valid
- load sample data to test the app

### The test will not run

If backtesting does not start:

- confirm both assets are selected
- check the time range
- make sure your data file is complete
- try a smaller date range first

## 📁 File tips

hedgevision works best when your files are clean and easy to read.

Use files with:
- clear column names
- matching dates
- no missing rows where possible
- one row for each time point

Good file names help too. Use names like:
- `AAPL_MSFT.csv`
- `pair_test_2024.csv`
- `sample_prices.csv`
